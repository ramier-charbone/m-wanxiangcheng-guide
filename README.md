# m-wanxiangcheng-guide

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 文件的集中存储与展示空间。

## 项目简介

本仓库主要收录各类独立 HTML 页面，这些页面可能涵盖说明文档、导航工具、信息整理、轻量应用等不同用途。每个页面均为自包含的 HTML 文件，无需依赖外部服务器环境即可直接运行。

通过本仓库，您可以：
- 快速访问和浏览已归档的 HTML 页面
- 了解页面的基本用途和结构
- 获取页面源代码以供参考或二次修改

## 目录说明

```
m-wanxiangcheng-guide/
├── index.html          # 仓库入口页面，列出所有归档页面链接
├── pages/              # 存放独立 HTML 页面文件的目录
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
└── assets/             # 页面引用的公共资源（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `index.html`：仓库的主页，通过简单列表或卡片形式展示所有归档页面。
- `pages/`：所有独立 HTML 页面均放置于此文件夹内，每个文件对应一个独立页面。
- `assets/`：存放页面中可能用到的公共样式、脚本或图片资源，便于复用和维护。

## 页面归档说明

每个归档页面通常包含以下信息：
- 页面标题与简短描述
- 页面功能或内容概述
- 使用方式或注意事项（如有）

归档页面均为静态 HTML，不依赖后端服务，可直接在浏览器中打开使用。部分页面可能引用外部资源（如字体、图标库），但不会主动收集用户数据或执行跟踪行为。

## 维护说明

- **新增页面**：将新 HTML 文件放入 `pages/` 目录，并在 `index.html` 中添加对应链接。
- **更新页面**：直接修改 `pages/` 中对应文件，并同步更新 `index.html` 中的描述（如有变化）。
- **删除页面**：移除 `pages/` 中的文件，并删除 `index.html` 中的对应链接。

欢迎通过 Issue 或 Pull Request 提出改进建议或报告问题。请确保提交的内容符合本仓库的归档范围（独立 HTML 页面，无恶意代码或侵权内容）。
